üîß Right to Repair & Open Source
If you buy something, you should be able to fix it.

This one feels pretty straightforward to me. If you own a device, you should be able to repair it ‚Äî and if you run software, you should at least be able to see how it works. The fact that that's even a debate is a bit strange when you think about it.

A lot of what I've learned about Linux and systems came from being able to read source code and poke around in things that were left open. That's much harder to do when software is locked down or hardware is designed to be thrown away instead of fixed. Open tools aren't just more interesting to work with ‚Äî they're genuinely better for learning, because you can actually see what's going on rather than just accepting that it does.
